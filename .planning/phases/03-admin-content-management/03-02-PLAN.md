<task type="auto">
  <name>Task 1: Create chapter validation schemas and server actions with audit logging</name>
  <files>src/lib/validations/chapter.ts, src/lib/actions/chapters.ts</files>
  <action>
  1. Create src/lib/validations/chapter.ts with Zod schemas:
     - `chapterSchema`: z.object({ name: z.string().min(1, "Numele capitolului este obligatoriu").max(200), description: z.string().max(1000).optional() })
     - Export the schema and inferred type `ChapterInput`

  2. Create src/lib/actions/chapters.ts with "use server" directive:

     - `getChaptersWithStats()`: Select all non-archived chapters ordered by sortOrder. For each chapter, count total questions, CS questions, and CM questions using a subquery or join. Return array of { id, name, description, sortOrder, questionCount, csCount, cmCount }. Filter out archived chapters (archivedAt IS NULL) and archived questions.

     - `createChapter(formData: FormData)`: Validate with chapterSchema. Get current max sortOrder, set new chapter's sortOrder to max+1. Insert chapter. Call logAudit with action="create", entityType="chapter". Revalidate /admin/chapters. Return { success: true, id } or { error }.

     - `updateChapter(id: string, formData: FormData)`: Validate with chapterSchema. Fetch old chapter for audit diff. Update chapter name, description, updatedAt. Call logAudit with action="update", changes showing old->new values. Revalidate /admin/chapters. Return { success: true } or { error }.

     - `archiveChapter(id: string)`: Set archivedAt to new Date(). Call logAudit with action="delete". Revalidate /admin/chapters. Return { success: true } or { error }.

     - `restoreChapter(id: string)`: Set archivedAt to null. Call logAudit with action="restore". Revalidate /admin/chapters. Return { success: true } or { error }.

     - `reorderChapters(orderedIds: string[])`: Use db.transaction to update sortOrder for each chapter based on array index. Call logAudit with action="reorder" for the first chapter (logging the full new order in changes). Revalidate /admin/chapters. Return { success: true } or { error }.

     ALL actions must call getCurrentAdmin() first to get the current user ID for audit logging and to enforce superadmin access.

  IMPORTANT: Use Drizzle's eq, isNull, asc, sql imports. For question counts, use Drizzle's sql template literal for count queries or separate select with where clause.
  </action>
