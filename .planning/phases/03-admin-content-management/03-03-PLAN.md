  <name>Task 1: Create question validation schemas and server actions with audit logging</name>
  <files>src/lib/validations/question.ts, src/lib/actions/questions.ts</files>
  <action>
  1. Create src/lib/validations/question.ts with Zod schemas:
     - `optionSchema`: z.object({ label: z.string(), text: z.string().min(1, "Textul optiunii este obligatoriu"), isCorrect: z.boolean() })
     - `questionSchema`: z.object({
         chapterId: z.string().uuid("Capitol invalid"),
         text: z.string().min(1, "Textul intrebarii este obligatoriu").max(2000),
         type: z.enum(["CS", "CM"]),
         options: z.array(optionSchema).length(5, "Trebuie exact 5 optiuni (A-E)"),
         sourceBook: z.string().optional(),
         sourcePage: z.string().optional(),
       })
     - Add `.refine()` for CS type: exactly 1 correct answer. For CM type: at least 2 correct answers.
     - Export schemas and inferred types

  2. Create src/lib/actions/questions.ts with "use server" directive:

     - `getQuestions(filters?: { chapterId?: string, type?: "CS" | "CM", search?: string, page?: number, pageSize?: number })`: Query non-archived questions with their options and chapter name. Support filtering by chapter, type, text search (ilike on question text). Paginate with default pageSize=20. Return { questions: Array<{ id, text, type, chapterName, sourceBook, sourcePage, optionCount, createdAt }>, total: number, page: number, pageSize: number }.

     - `getQuestionById(id: string)`: Fetch single question with all its options, ordered by label. Return full question data or null if not found/archived.

     - `createQuestion(data: QuestionInput)`: Validate with questionSchema. Use db.transaction: insert question, then insert 5 options with labels A-E. Call logAudit with action="create", entityType="question". Revalidate /admin/questions. Return { success: true, id } or { error }.

     - `updateQuestion(id: string, data: QuestionInput)`: Validate with questionSchema. Use db.transaction: fetch old question+options for audit diff, update question fields, delete old options and insert new ones (simpler than individual option updates). Call logAudit with action="update", changes showing what changed. Revalidate /admin/questions. Return { success: true } or { error }.

     - `archiveQuestion(id: string)`: Set archivedAt to new Date(). Call logAudit with action="delete". Revalidate /admin/questions. Return { success: true } or { error }.

     - `restoreQuestion(id: string)`: Set archivedAt to null. Call logAudit with action="restore". Revalidate /admin/questions.

     - `getChaptersForSelect()`: Return non-archived chapters as [{id, name}] for the chapter dropdown in the question form.

     ALL actions must call getCurrentAdmin() first. Use proper Drizzle imports (eq, isNull, ilike, sql, and, or, asc, desc, count).
  </action>

    <automated>npx tsc --noEmit</automated>
  </verify>
  <done>Question server actions handle CRUD with transaction-safe question+options operations. Zod validation enforces CS=1 correct, CM>=2 correct. All mutations audit-logged. Pagination and filtering supported.</done>
