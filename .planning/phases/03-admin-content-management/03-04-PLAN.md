<task type="auto">
  <name>Task 1: Build import validation and server actions for CSV/Excel import and export</name>
  <files>src/lib/validations/import.ts, src/lib/actions/import-export.ts</files>
  <action>
  1. Install PapaParse and ExcelJS:
     Run: `pnpm add papaparse exceljs`
     Run: `pnpm add -D @types/papaparse`

  2. Create src/lib/validations/import.ts:
     Define the CSV/Excel column structure (Claude's discretion per CONTEXT.md):
     ```
     Columns: id (optional, for upsert), chapter_name, question_text, type (CS/CM), option_a, option_b, option_c, option_d, option_e, correct_answers (comma-separated: "A,C,E"), source_book (optional), source_page (optional)
     ```

     - `importRowSchema`: Zod schema for a single import row:
       - id: z.string().uuid().optional() (for upsert)
       - chapter_name: z.string().min(1)
       - question_text: z.string().min(1)
       - type: z.enum(["CS", "CM"])
       - option_a through option_e: z.string().min(1)
       - correct_answers: z.string().min(1) -- validate format (comma-separated A-E letters)
       - source_book: z.string().optional()
       - source_page: z.string().optional()
     - Add refinement: for CS, correct_answers must have exactly 1 letter; for CM, at least 2.
     - Export types: ImportRow, ValidatedImportRow, ImportError

  3. Create src/lib/actions/import-export.ts with "use server" directive:

     **Import action:**
     - `importQuestions(rows: ImportRow[])`: Server action that:
       a. Calls getCurrentAdmin()
       b. Validates each row against importRowSchema, collecting errors with row numbers
       c. Resolves chapter_name -> chapter_id (look up chapters by name, case-insensitive)
       d. For rows with unrecognized chapter names: add validation error
       e. For valid rows: use db.transaction to batch insert/upsert
         - If row has `id` and question exists: update question + delete old options + insert new options
         - If row has no `id` or question doesn't exist: insert new question + options
       f. Call logAudit for each imported question (action="create" or "update")
       g. Revalidate /admin/questions
       h. Return { imported: number, updated: number, errors: Array<{row: number, message: string}> }
       BATCH SIZE: Process in batches of 50 rows per transaction to avoid timeout.

     **Export actions:**
     - `exportQuestionsCSV(chapterId?: string)`: Server action that:
       a. Calls getCurrentAdmin()
       b. Queries all non-archived questions (filtered by chapter if provided) with their options and chapter names
       c. Transforms to the import format (same columns as import for round-trip compatibility)
       d. Returns the data as an array of objects (client will use PapaParse.unparse to create CSV with UTF-8 BOM)

     - `exportQuestionsExcel(chapterId?: string)`: Server action that:
       a. Calls getCurrentAdmin()
       b. Queries same data as CSV export
       c. Uses ExcelJS to create a Workbook with a "Questions" worksheet
       d. Set columns: id, chapter_name, question_text, type, option_a-e, correct_answers, source_book, source_page
       e. Add header row with bold formatting
       f. Add data rows
       g. Return the buffer as base64 string (client will decode and trigger download)

     IMPORTANT: For CSV export, the client-side download must add UTF-8 BOM ("\uFEFF") before the CSV content so Excel opens it with correct Romanian diacritics. For Excel export, ExcelJS handles encoding internally.
  </action>
